## GadgetBoy POS v0.2.83

### Reporting and batch-out fixes
- Fixed EOD and scheduled report payment math so collected totals follow payment history and stable payment dates instead of generic edit timestamps.
- Fixed batch-out cash/card totals to classify non-cash payments more accurately and stop reopened records from skewing totals.
- Cleaned up EOD activity summaries so collected values are shown correctly for work orders and sales.

### Notifications and calendar fixes
- Fixed calendar-driven notifications so part order and delivery reminders stop leaving behind stale entries after calendar edits.
- Notification keys for consultations, parts deliveries, and events are now stable across calendar entry replacements.

### Startup behavior
- Removed the startup update gate so the main app opens directly after the data-path gate.

### Payment history hardening
- Preserved legacy payment-history normalization for work orders and sales so historical collections continue to report accurately after later edits.
